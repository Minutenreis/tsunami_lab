 \hypertarget{md_submodules_Catch2_docs_command-line_autotoc_md51}{}\doxysubsection{Specifying which tests to run}\label{md_submodules_Catch2_docs_command-line_autotoc_md51}

\begin{DoxyPre}<test-\/spec> ...\end{DoxyPre}


Test cases, wildcarded test cases, tags and tag expressions are all passed directly as arguments. Tags are distinguished by being enclosed in square brackets.

If no test specs are supplied then all test cases, except \char`\"{}hidden\char`\"{} tests, are run. A test is hidden by giving it any tag starting with (or just) a period ({\ttfamily .}) -\/ or, in the deprecated case, tagged {\ttfamily \mbox{[}hide\mbox{]}} or given name starting with `'./\textquotesingle{}{\ttfamily . To specify hidden tests from the command line}{\ttfamily \mbox{[}.\mbox{]}}{\ttfamily or}{\ttfamily \mbox{[}hide\mbox{]}}\`{} can be used {\itshape regardless of how they were declared}.

Specs must be enclosed in quotes if they contain spaces. If they do not contain spaces the quotes are optional.

Wildcards consist of the {\ttfamily $\ast$} character at the beginning and/or end of test case names and can substitute for any number of any characters (including none).

Test specs are case insensitive.

If a spec is prefixed with {\ttfamily exclude\+:} or the {\ttfamily $\sim$} character then the pattern matches an exclusion. This means that tests matching the pattern are excluded from the set -\/ even if a prior inclusion spec included them. Subsequent inclusion specs will take precedence, however. Inclusions and exclusions are evaluated in left-\/to-\/right order.

Test case examples\+:


\begin{DoxyPre}thisTestOnly            Matches the test case called, 'thisTestOnly'
"this test only"        Matches the test case called, 'this test only'
these*                  Matches all cases starting with 'these'
exclude:notThis         Matches all tests except, 'notThis'
\string~notThis                Matches all tests except, 'notThis'
\string~*private*              Matches all tests except those that contain 'private'
a* \string~ab* abc             Matches all tests that start with 'a', except those that
                        start with 'ab', except 'abc', which is included
\begin{DoxyEnumerate}
\item [\#somefile]          Matches all tests from the file 'somefile.cpp'

\end{DoxyEnumerate}\end{DoxyPre}



\begin{DoxyPre}Names within square brackets are interpreted as tags.
A series of tags form an AND expression whereas a comma-\/separated sequence forms an OR expression. e.g.:\end{DoxyPre}



\begin{DoxyPre}
\begin{DoxyPre}[one][two],[three]\end{DoxyPre}
 This matches all tests tagged {\ttfamily \mbox{[}one\mbox{]}} and {\ttfamily \mbox{[}two\mbox{]}}, as well as all tests tagged {\ttfamily \mbox{[}three\mbox{]}}\end{DoxyPre}



\begin{DoxyPre}Test names containing special characters, such as {\ttfamily ,} or {\ttfamily [} can specify them on the command line using {\ttfamily \textbackslash{}}.
{\ttfamily \textbackslash{}} also escapes itself.\end{DoxyPre}

\begin{DoxyPre}
\begin{DoxyPre}-\/-\/benchmark-\/confidence-\/interval <confidence-\/interval>\end{DoxyPre}
\end{DoxyPre}



\begin{DoxyPre}> \href{https://github.com/catchorg/Catch2/issues/1616}{\texttt{ Introduced}} in Catch 2.9.0.\end{DoxyPre}



\begin{DoxyPre}The confidence-\/interval is used for statistical bootstrapping on the samples to
calculate the upper and lower bounds of mean and standard deviation.
Must be between 0 and 1 and defaults to 0.95.\end{DoxyPre}

\begin{DoxyPre}\label{_run-section}%

\end{DoxyPre}
\hypertarget{md_submodules_Catch2_docs_command-line_autotoc_md73}{}\doxysubsection{Specify the section to run}\label{md_submodules_Catch2_docs_command-line_autotoc_md73}

\begin{DoxyPre}
\begin{DoxyPre}-\/c, -\/-\/section <section name>\end{DoxyPre}
\end{DoxyPre}



\begin{DoxyPre}To limit execution to a specific section within a test case, use this option one or more times.
To narrow to sub-\/sections use multiple instances, where each subsequent instance specifies a deeper nesting level.\end{DoxyPre}



\begin{DoxyPre}E.g. if you have:\end{DoxyPre}



\begin{DoxyPre}
\begin{DoxyPre}
TEST\_CASE( "Test" ) \{
  SECTION( "sa" ) \{
    SECTION( "sb" ) \{
      /*...*/
    \}
    SECTION( "sc" ) \{
      /*...*/
    \}
  \}
  SECTION( "sd" ) \{
    /*...*/
  \}
\}
\end{DoxyPre}
\end{DoxyPre}



\begin{DoxyPre}Then you can run {\ttfamily sb} with:

\begin{DoxyPre}./MyExe Test -\/c sa -\/c sb\end{DoxyPre}
\end{DoxyPre}



\begin{DoxyPre}Or run just {\ttfamily sd} with:

\begin{DoxyPre}./MyExe Test -\/c sd\end{DoxyPre}
\end{DoxyPre}



\begin{DoxyPre}To run all of {\ttfamily sa}, including {\ttfamily sb} and {\ttfamily sc} use:

\begin{DoxyPre}./MyExe Test -\/c sa\end{DoxyPre}
\end{DoxyPre}



\begin{DoxyPre}There are some limitations of this feature to be aware of:
\begin{DoxyItemize}
\item Code outside of sections being skipped will still be executed -\/ e.g. any set-\/up code in the TEST\_CASE before the
start of the first section.
\item At time of writing, wildcards are not supported in section names.
\item If you specify a section without narrowing to a test case first then all test cases will be executed 
(but only matching sections within them).
\end{DoxyItemize}\end{DoxyPre}

\end{DoxyPre}
\hypertarget{md_submodules_Catch2_docs_command-line_autotoc_md74}{}\doxysubsection{Filenames as tags}\label{md_submodules_Catch2_docs_command-line_autotoc_md74}

\begin{DoxyPre}
\begin{DoxyPre}-\/\#, -\/-\/filenames-\/as-\/tags\end{DoxyPre}
\end{DoxyPre}



\begin{DoxyPre}When this option is used then every test is given an additional tag which is formed of the unqualified 
filename it is found in, with any extension stripped, prefixed with the {\ttfamily \#} character.\end{DoxyPre}



\begin{DoxyPre}So, for example,  tests within the file {\ttfamily \string~\textbackslash{}Dev\textbackslash{}MyProject\textbackslash{}Ferrets.cpp} would be tagged {\ttfamily [\#Ferrets]}.\end{DoxyPre}

\hypertarget{md_submodules_Catch2_docs_command-line_autotoc_md75}{}\doxysubsection{Override output colouring}\label{md_submodules_Catch2_docs_command-line_autotoc_md75}

\begin{DoxyPre}
\begin{DoxyPre}-\/-\/use-\/colour <yes|no|auto>\end{DoxyPre}
\end{DoxyPre}



\begin{DoxyPre}Catch colours output for terminals, but omits colouring when it detects that
output is being sent to a pipe. This is done to avoid interfering with automated
processing of output.\end{DoxyPre}



\begin{DoxyPre}{\ttfamily -\/-\/use-\/colour yes} forces coloured output, {\ttfamily -\/-\/use-\/colour no} disables coloured
output. The default behaviour is {\ttfamily -\/-\/use-\/colour auto}.\end{DoxyPre}
